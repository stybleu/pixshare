API PixShare

API d’upload de fichiers avec authentification par clé API.

Authentification

Toutes les routes protégées utilisent l’un des headers suivants : 

Authorization: Bearer VOTRE_CLE_API
X-API-Key: VOTRE_CLE_API
Authorization: Bearer ... et X-API-Key sont tous les deux acceptés. Si tu utilises Authorization, le mot Bearer est obligatoire.

1. Vérifier son compte API

Route : GET /api/account

curl -X GET {{ request.host_url[:-1] }}/api/account \
-H "Authorization: Bearer VOTRE_CLE_API"

Retourne les informations de la clé, les limites et les options autorisées.

Type de réponse JSON :

{
  "success": true,
  "status": 200,
  "data": {
    "key_name": "string",
    "is_active": true,
    "max_file_size_mb": 10,
    "allow_permanent": false,
    "default_lifetime_minutes": 10,
    "allowed_lifetimes": [5, 10, 20, 30, 60],
    "limits": {
      "remaining_total": 0,
      "remaining_today": 0,
      "used_total": 0,
      "used_today": 0,
      "max_total": 0,
      "max_per_day": 0
    }
  }
}

2. Upload d’un fichier

Route : POST /api/upload

  • file : fichier à envoyer
  • expiration : durée en minutes

Exemple (PC) :

curl -X POST {{ request.host_url[:-1] }}/api/upload \
-H "Authorization: Bearer VOTRE_CLE_API" \
-F "file=@image.jpg" \
-F "expiration=10"

Exemple (Android / Termux) :

curl -X POST {{ request.host_url[:-1] }}/api/upload \
-H "Authorization: Bearer VOTRE_CLE_API" \
-F "file=@/storage/emulated/0/Download/image.jpg" \
-F "expiration=10"
Le champ attendu est expiration, pas lifetime.

      "used_today": 0,
      "max_total": 0,

3. Lire un fichier

Route : GET /api/file/<id>

curl -X GET {{ request.host_url[:-1] }}/api/file/ID \
-H "Authorization: Bearer VOTRE_CLE_API"

Type de réponse JSON :

{
  "success": true,
  "status": 200,
  "data": {
    "id": "string",
    "title": "string",
    "filename": "string",
    "original_filename": "string",
    "url_viewer": "string",
    "url": "string",
    "display_url": "string",
    "delete_url": "string",
    "mime": "string",
    "extension": "string",
    "size": 0,
    "uploaded_at": "ISO-8601 string",
    "expiration": "ISO-8601 string | null",
    "is_permanent": false,
    "uploader_api_key_name": "string"

}

4. Supprimer un fichier par clé API

Route : DELETE /api/file/<id>

curl -X DELETE {{ request.host_url[:-1] }}/api/file/ID \
-H "Authorization: Bearer VOTRE_CLE_API"
Doit utiliser la même clé API que celle utilisée pour l’upload.

Type de réponse JSON :

{
  "success": true,
  "status": 200,
  "data": {
    "id": "string",
    "deleted": true
  }
}

5. Suppression par lien public

Route : GET /api/delete/<token> ou DELETE /api/delete/<token> selon ton implémentation

Important : ce lien n’est valide que si ton backend stocke bien un delete_token pour chaque fichier. Si le token n’existe pas encore, delete_url peut être vide ou incomplet.

Type de réponse JSON attendu :

{
  "success": true,
  "status": 200,
  "data": {
    "id": "string",
    "deleted": true
  }
}

Exemple réel de réponse d’upload

{

  "status": 200,
  "data": {

    "display_url": "{{ request.host_url[:-1] }}/api/raw/iI4c5mudFEXGfqn.png",

Attention : si delete_url vaut seulement {{ request.host_url[:-1] }}/api/delete/, cela signifie généralement que le delete_token n’a pas été enregistré côté backend.

Signification des champs

Champ Type Description
success bool Indique si la requête a réussi.
status int Code HTTP renvoyé par l’API.
data.id string Identifiant unique du fichier.
data.title string Nom d’origine affichable.
data.filename string Nom réellement stocké sur le serveur.
data.original_filename string Nom du fichier envoyé par le client.
data.url string Lien direct brut vers le fichier.
data.display_url string Lien direct affichable, identique à url dans ton implémentation actuelle.
data.url_viewer string Page HTML de visualisation du fichier.
data.delete_url string Lien public de suppression par token. Peut être vide/incomplet si aucun token n’est enregistré.
data.mime string Type MIME détecté.
data.extension string Extension du fichier sans point.
data.size int Taille du fichier en octets.
data.uploaded_at string Date d’upload au format ISO 8601 UTC.
data.expiration string | null Date d’expiration au format ISO 8601 UTC, ou null si permanent.
data.is_permanent bool Indique si le fichier est permanent.
data.uploader_api_key_name string Nom lisible de la clé API ayant envoyé le fichier.
data.limits object État des quotas après l’upload.
